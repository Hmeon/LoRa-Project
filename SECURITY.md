# Security Policy

## Supported scope
This repository is a research/field-experiment runtime and does not ship a production service.

## Reporting a vulnerability
- Prefer private disclosure via GitHub Security Advisories (if enabled for this repo).
- If that is not available, open a GitHub issue with minimal details and clearly mark it as a security report.

## Sensitive data
- Do not include device IDs, addresses, keys, or private datasets in issues/PRs.
- Use `configs/examples/` templates for configuration examples.
